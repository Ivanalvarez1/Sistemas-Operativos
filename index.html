Explorando los Sistemas Operativos

Sistemas Operativos Móviles

Android

Android es un sistema operativo basado en el núcleo de Linux y en componentes de software de código abierto, desarrollado principalmente por Google, destaca por ofrecer una interfaz altamente personalizable mediante launchers, widgets y otras aplicaciones, Android está estructurado en varias capas: Núcleo Linux, Runtime y bibliotecas, Framework de aplicaciones y Aplicaciones base

iOS

IOS es el sistema operativo desarrollado por Apple inc. es exclusivo para iPhone, este fue presentado por primera vez en 2007 revolucionando la industria de la telefonía móvil por su interfaz táctil intuitiva y su diseño elegante. IOS destaca por su robusto sistema de seguridad dado que funciona en un sistema cerrado. Esto permite una integración optimizada y una mayor proteccion contra malware

Sistemas Operativos de PC

Windows

El sistema operativo de Microsoft, ampliamente utilizado en PCs. Conocido por su interfaz gráfica amigable y su compatibilidad con una gran variedad de software y hardware. Las principales características de este sistema operativo son: la interfaz gráfica de usuario (GUI), la gran compatibilidad, el sistema multitareas y el sistema de archivos (NTFS).

MacOS

El sistema operativo de Apple para computadoras Mac. Se destaca por su diseño elegante, su integración con otros dispositivos de Apple y su enfoque en la seguridad y la privacidad del usuario. La Mac App Store ofrece una gran selección de aplicaciones de alta calidad.

Linux

Un sistema operativo de código abierto basado en Unix. Es conocido por su estabilidad y seguridad, y es ampliamente utilizado en servidores y sistemas embebidos. Linux tiene una gran comunidad de desarrolladores y una vasta cantidad de distribuciones, como Ubuntu, Fedora y CentOS.

Automotrices

Los sistemas operativos para automóviles están diseñados para gestionar tanto el hardware como el software de los vehículos, permitiendo la interacción entre diversos módulos como infoentretenimiento, navegación, sensores, y sistemas avanzados de asistencia al conductor (ADAS). Estos sistemas optimizan la experiencia del usuario y la eficiencia del vehículo, integrando aplicaciones, datos y comunicación con diferentes dispositivos.

Sistemas Operativos para Automóviles

  • QNX: Muy utilizado por su fiabilidad,tiempo real y certificaciones de seguridad funcional
  • Android Automotive: Basado en Android, permite la personalización y la integración de aplicaciones móviles
  • Linux: Algunas distribuciones de Linux se adaptan para su uso en vehículos, ofreciendo flexibilidad y personalización
  • Windows Embedded Automotive: Una versión de Windows diseñada específicamente para sistemas de infoentretenimiento en vehículos
  • Otros Sistemas propietarios: Tesla OS, MBUX (Mercedes-Benz), Licid UX y otros desarrolladores internos de grandes fabricantes

Breve Historia

Los sistemas operativos automotrices surgieron en los años 90, evolucionando hacia estándares abiertos y multiplataforma como QNX, Linux y Android Automotive. Actualmente, están enfocados en soportar las demandas de conectividad, autonomía y experiencia de usuario de los vehículos modernos, estos sitemas son esenciales para permitir funciones avanzadas como conducción semiautónoma integración con smartphones, actualización remota y personalización de la experiencia del usuario dentro del automovil.

Clústers

Un clúster de computadoras es un conjunto de equipos conectados entre sí que trabajan de manera conjunta como un solo sistema, coordinando recursos para lograr mayor potencia de cómputo, disponibilidad y eficiencia. Los sistemas operativos más empleados en clústers suelen ser variantes de Linux, como CentOS, Ubuntu y Red Hat Enterprise, así como sistemas diseñados específicamente para computación de alto rendimiento (HPC).

Sistemas Operativos para Clústers

  • Linux y variantes especificadas para HPC: Linux domina el mercado con distribuciones especializadas como Ubuntu Server, CentOS, Red Hat Enterprise Linux y entornos personalizados como Cray Linux y HPE Cray OS.
  • Sistemas operativos especializados y minimalistas: Existen sistemas como ZeptoOS, FusedOS, Hobbes, NIX, IHK/MCKernel o mOS que son proyectos de investigación o desarrollos experimentales enfocados en modularidad, seguridad eficiencia y escalabilidad.
  • Windows Server 2025: Aunque con menor presencia, Windows Server tiene integraciones avanzadas para clústers de conmutación por error y administración moderna en entornos empresariales.

Breve Historia

Los clústers surgieron a mediados del siglo XX impulsados por la necesidad de aumentar el rendimiento y disponibilidad de los sistemas computacionales sin recurrir a supercomputadoras costosas. Con el auge de hardware estándar y software de código abierto (especialmente linux), su adopción se extendió desde universidades hasta empresas privadas y servicios en la nube. Hoy los clústers son clave en investigación científica, cálculos financieros, big data y servicios criticos en linea.

Embebidos

Los sistemas embebidos son dispositivos con recursos limitados y orientados a realizar funciones especificas, ademas de ser eficientes en el uso de la memoria y energía, y a menudo funcionan en tiempo real para controlar dispositivos integrados en productos como automóviles, electrodomesticos, dispositivos médicos y maquinaria industrial.

Sistemas Operativos Embebidos

  • Sistemas operativos en tiempo real (RTOS): Diseñados para responder en tiempos limitados y deterministas; ejemplos incluyen FreeRTOS, VxWorks, QNX y ThreadX.
  • Linux embebido: Variantes de Linux adaptadas para sistemas embebidos como Yocto, Buildroot Y Android Things, que ofrecen mayor capacidad y flexibilidad con recursos moderados.
  • Sistemas propietarios y minimalistas: Muchas compañias desarrollan sistemas operativos embebidos a medida para aplicaciones especificas.
  • Sistemas embebidos basados en microcontroladores: El sistema operativo puede ser mínimamente un bucle de ejecución o sin sistema operativo, programado directamente en ensamblador o C.

Breve Historia

Los primeros sistemas embebidos aparecieron en 1980, inicialmente para controlar hardware industrial y electrodomésticos. Con el crecimiento de la electrónica digital y la miniaturización de procesadores y memoria, su uso se extendió a numerosos dispositivos electrónicos y sistemas automatizados. Hoy, los sistemas operativos embebidos abarcan desde soluciones muy sencillas hasta plataformas complejas en IoT, automotriz, telecomunicaciones y domótica.

Mainframes

Los mainframes, o grandes computadoras centrales, son equipos de alto rendimiento diseñados para procesar cantidades masivas de datos de forma continua y con una fiabilidad extrema. Sus sistemas operativos son, por tanto, algunos de los más robustos, seguros y complejos del mundo de la computación. Están especializados en el procesamiento transaccional a gran escala (como operaciones bancarias, reservas de aerolíneas o procesamiento de censos) y en la gestión de bases de datos gigantescas.

Sistemas Operativos para Mainframes

  • z/OS: Es el sistema operativo insignia y más avanzado para los mainframes de IBM. z/OS es el descendiente directo del OS/360, el sistema operativo que inauguró la era de la computación moderna en la década de 1960. Está diseñado para ofrecer el máximo nivel de estabilidad, seguridad y capacidad de procesamiento.
  • z/VSE (Virtual Storage Extended): Un sistema operativo más ligero y sencillo que z/OS, orientado a pequeñas y medianas empresas. Aunque no tiene todas las capacidades avanzadas de z/OS, z/VSE sigue siendo muy potente y es adecuado para muchas aplicaciones empresariales tradicionales.
  • z/VM (Virtual Machine): z/VM es un hipervisor de alto rendimiento. Su función principal no es ejecutar aplicaciones directamente, sino crear y gestionar miles de máquinas virtuales en un solo mainframe. Cada máquina virtual puede ejecutar su propio sistema operativo, que puede ser otra instancia de z/OS, z/VSE o, muy comúnmente, Linux.
  • Linux on IBM Z: No es un sistema operativo de IBM, sino distribuciones estándar de Linux (como Red Hat, SUSE, Ubuntu) que han sido compiladas y optimizadas para ejecutarse en la arquitectura de hardware del mainframe. Típicamente, se ejecuta dentro de máquinas virtuales gestionadas por z/VM.

Breve Historia

  • Procesamiento por Lotes (60s-70s): Los primeros sistemas, como OS/360, estaban diseñados para ejecutar trabajos uno tras otro a partir de tarjetas perforadas.
  • Interactividad y Virtualización (70s-80s): Sistemas como MVS y VM introdujeron el tiempo compartido (múltiples usuarios interactuando a través de terminales) y la virtualización, permitiendo una utilización mucho más eficiente del hardware.
  • Conectividad (90s): Los sistemas operativos de mainframe adoptaron los estándares de redes como TCP/IP, abriéndose al mundo de la computación cliente-servidor.
  • Modernización y Código Abierto (2000s - Presente): Con la llegada de z/OS, z/VM y, sobre todo, Linux on Z, los mainframes se transformaron en servidores empresariales modernos, capaces de ejecutar tanto las aplicaciones legacy críticas como las nuevas cargas de trabajo basadas en Java, contenedores y APIs para la nube híbrida.

Redes

Los sistemas operativos de red (NOS - Network Operating System) son el software especializado que se ejecuta en dispositivos de red como routers, switches y firewalls. Su función principal es gestionar el hardware del dispositivo y proporcionar las funcionalidades necesarias para dirigir el tráfico de datos a través de una red de manera eficiente, segura y fiable. A diferencia de los sistemas operativos de propósito general (como Windows o macOS), los NOS están altamente optimizados para tareas de red: procesar paquetes de datos a velocidades de línea, implementar protocolos de enrutamiento complejos y aplicar políticas de seguridad.

Sistemas Operativos de Red (NOS)

  • Cisco IOS (Internetwork Operating System): Es posiblemente el sistema operativo de red más conocido y extendido del mundo, desarrollado por Cisco Systems. Durante décadas, ha sido el estándar de facto en la industria y la base sobre la cual se han formado la mayoría de los ingenieros de redes.
  • Juniper Junos OS: Es el sistema operativo desarrollado por Juniper Networks, el principal competidor de Cisco. Junos OS es conocido por su diseño limpio y su arquitectura superior. A diferencia del IOS original, Junos fue diseñado desde el principio sobre un núcleo de software de código abierto (FreeBSD, un sistema tipo Unix).
  • Arista EOS (Extensible Operating System): Es un sistema operativo de red relativamente moderno desarrollado por Arista Networks, una empresa fundada por veteranos de la industria. EOS está construido sobre un núcleo de Linux estándar y sin modificar, lo que lo hace extremadamente abierto y programable. Está enfocado principalmente en los centros de datos a gran escala y la computación en la nube.

Breve Historia

  • Era Monolítica (80s-90s): Los primeros sistemas, como el IOS original, eran un único bloque de software fuertemente acoplado al hardware. Eran rápidos pero difíciles de actualizar y un fallo en un componente podía detener todo el sistema.
  • Era Modular (Finales de los 90 - 2000s): Sistemas como Junos introdujeron la modularidad, separando los procesos para mejorar la estabilidad y la fiabilidad. El software se basó en núcleos de sistemas operativos robustos como Unix/BSD.
  • Era de la Programabilidad y el Desacoplamiento (2010s - Presente): La llegada de la nube y los centros de datos masivos impulsó la necesidad de automatización. Sistemas como EOS, construidos sobre Linux, abrieron los dispositivos de red a los desarrolladores y a las herramientas de DevOps. Surge el concepto de desacoplamiento, donde el sistema operativo de red podría, teóricamente, ejecutarse en hardware de diferentes fabricantes (redes abiertas).

Relojes

Los sistemas operativos para relojes inteligentes son la plataforma de software diseñada para ejecutarse en dispositivos portátiles (wearables) de muñeca. A diferencia de los sistemas para smartphones, están optimizados para pantallas pequeñas, interacciones rápidas y un bajo consumo de energía. Su función principal es conectar el reloj al smartphone, gestionar notificaciones, monitorizar la salud y la actividad física, y ejecutar aplicaciones ligeras.

Sistemas Operativos para Relojes

  • Apple watchOS: Es el sistema operativo desarrollado por Apple exclusivamente para su línea de productos Apple Watch. Se caracteriza por su profunda integración con el ecosistema de Apple (iPhone, iCloud, Apple Health, etc.), una interfaz de usuario pulida y un fuerte enfoque en la salud, el fitness y la comunicación. Solo es compatible con el iPhone.
  • Google Wear OS: Es la plataforma de Google para relojes inteligentes y otros wearables, diseñada para funcionar en dispositivos de múltiples fabricantes (como Samsung, Google, Fossil, TicWatch). Originalmente conocido como Android Wear, su objetivo es llevar el poder del Asistente de Google, las notificaciones de Android y los servicios de Google a la muñeca.
  • Tizen OS for Wearables: Tizen era el sistema operativo de código abierto, basado en Linux, desarrollado y utilizado por Samsung para sus relojes inteligentes (línea Galaxy Watch) y otros dispositivos. Era conocido por su excelente rendimiento, buena duración de la batería y su bisel giratorio para la navegación.
  • HarmonyOS (para Wearables): Es el sistema operativo desarrollado por Huawei, diseñado para funcionar en un ecosistema de dispositivos interconectados (teléfonos, tablets, televisores y relojes). En sus smartwatches, HarmonyOS se enfoca en una experiencia fluida, una buena gestión de la batería

Breve Historia

  • Primeros Intentos (2000s): Los primeros relojes inteligentes, como el IBM WatchPad (2001) y el Microsoft SPOT (2004), tenían sistemas operativos muy básicos y limitados, principalmente para mostrar notificaciones y datos simples.
  • Era de Android Wear (2014-2018): Google lanzó Android Wear en 2014, marcando el inicio de una era más seria para los relojes inteligentes. La plataforma permitió a múltiples fabricantes crear dispositivos con capacidades avanzadas, aunque la adopción fue lenta debido a problemas de rendimiento y duración de la batería.
  • Consolidación y Competencia (2018-Presente): En 2018, Google relanzó la plataforma como Wear OS, buscando mejorar la experiencia del usuario y atraer a más fabricantes. Mientras tanto, Apple consolidó su posición con watchOS, que se convirtió en el sistema operativo líder en el mercado de relojes inteligentes. Samsung también migró sus relojes a Wear OS en 2021, abandonando Tizen.
  • Futuro (Presente y Más Allá): La competencia entre watchOS y Wear OS continúa, con ambos sistemas operativos evolucionando rápidamente para ofrecer mejores experiencias de usuario, mayor integración con otros dispositivos y nuevas funcionalidades centradas en la salud y el bienestar.

Virtualización

Los sistemas operativos de virtualización, más conocidos como hipervisores, son plataformas de software que permiten crear y ejecutar máquinas virtuales (VMs). Una máquina virtual es una simulación completa de un sistema informático (CPU, memoria, disco, red) sobre la cual se puede instalar un sistema operativo completo (llamado "sistema operativo invitado"). Existen dos tipos principales de hipervisores: Tipo 1 (Nativo o "Bare Metal") y Tipo 2 (Alojado o "Hosted").

Sistemas Operativos de Virtualización Tipo 1

  • VMware vSphere ESXi: Es el líder indiscutible del mercado de la virtualización empresarial, desarrollado por VMware. ESXi es un hipervisor de tipo "bare metal" que se instala directamente en el hardware del servidor.
  • Microsoft Hyper-V: Es la solución de virtualización de Microsoft y el principal competidor de VMware. Hyper-V viene integrado como un "rol" en las versiones de Windows Server y también está disponible como un producto independiente y gratuito (Hyper-V Server).
  • KVM (Kernel-based Virtual Machine): Es la solución de virtualización de código abierto integrada en el núcleo (kernel) de Linux. KVM convierte el propio kernel de Linux en un hipervisor de Tipo 1. No es un sistema operativo independiente, sino una funcionalidad que puede ser activada en cualquier distribución moderna de Linux (como Red Hat, Ubuntu, SUSE)
  • Citrix Hypervisor (anteriormente XenServer): Es un hipervisor de Tipo 1 basado en el proyecto de código abierto Xen. Citrix Hypervisor está optimizado para entornos de virtualización de escritorios y aplicaciones (VDI), siendo un componente clave de la suite de productos Citrix Virtual Apps and Desktops.

Breve Historia

  • Años 60: Los conceptos de virtualización nacen en los mainframes de IBM (con sistemas como CP/CMS) para compartir los carísimos recursos de una sola máquina entre múltiples usuarios.
  • Años 90: La proliferación de servidores x86 baratos crea el problema del "server sprawl" (un servidor, una aplicación), resultando en una ineficiencia masiva.
  • 1998-2001: VMware revoluciona el mercado al llevar la virtualización de los mainframes al mundo x86 con productos como Workstation y el hipervisor ESX Server, dando inicio a la era del centro de datos moderno.
  • 2007-2008: La tecnología se masifica. KVM se integra en el kernel de Linux y Microsoft lanza Hyper-V, convirtiendo la virtualización en una característica estándar y no en un producto de nicho.
  • 2010s - Presente: La virtualización se convierte en la base del Cloud Computing. La evolución continúa con los contenedores (Docker), que virtualizan a nivel de sistema operativo en lugar de hardware, y la orquestación a gran escala con Kubernetes.

Workstation

Una workstation o estación de trabajo es una computadora de alto rendimiento, diseñada para un solo usuario con tareas profesionales exigentes como diseño gráfico (CAD/CAM), edición de video, animación 3D, cálculo científico, desarrollo de software avanzado o análisis financiero. Por lo tanto, sus sistemas operativos están optimizados para estabilidad bajo cargas pesadas, gestión de grandes cantidades de memoria RAM, soporte para hardware especializado (como GPUs profesionales y múltiples CPUs).

Sistemas Operativos para Workstations

  • Windows for Workstations (Pro/Enterprise): Es una edición de gama alta del sistema operativo de escritorio de Microsoft, actualmente Windows 11 Pro for Workstations. Está específicamente diseñada para aprovechar el hardware de estaciones de trabajo modernas y satisfacer las necesidades de cargas de trabajo de misión crítica.
  • Apple macOS: Es el sistema operativo de Apple para su línea de computadoras Mac. Históricamente, macOS (anteriormente Mac OS X) ha sido la plataforma preferida en muchas industrias creativas como el diseño gráfico, la edición de video y la producción de audio, gracias a su interfaz de usuario pulida, su estabilidad (basada en Unix) y un ecosistema de software creativo muy maduro.
  • Linux (Distribuciones para Workstation): Linux es una opción extremadamente popular para workstations, especialmente en los campos del desarrollo de software, la investigación científica, la ingeniería y los efectos especiales (VFX). No es un único SO, sino un ecosistema de distribuciones (como Ubuntu, Fedora Workstation, Red Hat Enterprise Linux Workstation, SUSE Linux Enterprise Desktop).

Breve Historia

  • (Años 80 y 90): La era original de las workstations fue dominada por costosas máquinas de fabricantes como Sun Microsystems (Solaris), Silicon Graphics (IRIX), HP (HP-UX) y IBM (AIX). Estos sistemas operativos UNIX propietarios estaban décadas por delante de los PCs de la época en gráficos, redes y multitarea.
  • (Mediados de los 90): Microsoft Windows NT ofreció capacidades "suficientemente buenas" (multiprocesamiento, sistema de archivos robusto) en hardware x86 mucho más barato. Esto comenzó a erosionar el mercado de las costosas workstations UNIX.
  • (2000s): El mercado se polarizó. Mac OS X, con su base UNIX y su enfoque en la creatividad, capturó a los diseñadores y editores de video. Linux se convirtió en el rey del desarrollo de software, la ciencia y los efectos especiales. Windows dominó en el CAD de ingeniería, la arquitectura y el sector corporativo general.
  • Las diferencias de rendimiento se han acortado. La elección de un SO de workstation hoy depende más del ecosistema de software, el flujo de trabajo y la integración con el hardware específico

Software Desarrollo

Un Entorno de Desarrollo Integrado es una aplicación de software que consolida las herramientas básicas que los desarrolladores necesitan para escribir y probar software. Un IDE contiene, como mínimo, un editor de código fuente, un compilador o intérprete, y un depurador, a los que se accede a través de una única interfaz gráfica de usuario (GUI), en cambio un framework, en el contexto del desarrollo de software, es una estructura conceptual y tecnológica de soporte definido, con módulos de software concretos, que sirve de base para la organización y desarrollo de software. Proporciona una arquitectura predefinida, un conjunto de bibliotecas y componentes reutilizables, y un flujo de control invertido (Inversion of Control), donde el propio framework es el que invoca el código del desarrollador.

IDE's y sus lenguajes

  • Visual Studio Code (VS Code): Es un editor de código fuente muy ligero y extensible, pero con las extensiones adecuadas funciona como un IDE completo. Es extremadamente popular por su versatilidad. Los lenguajes más utilizados son: JavaScript, TypeScript, Python, Go, Java, y desarrollo web en general (HTML, CSS).
  • Visual Studio: Es el IDE más robusto de Microsoft y una opción muy potente para el desarrollo en el ecosistema de Windows y .NET. Los lenguajes más Utilizados son Principalmente C# y Visual Basic para el desarrollo con el framework .NET. También tiene un soporte excelente para C++ (especialmente para desarrollo de aplicaciones y videojuegos en Windows) y Python.
  • JetBrains IDEs (IntelliJ IDEA, PyCharm, WebStorm): JetBrains es una compañía que desarrolla varios IDEs, cada uno especializado en un lenguaje o ecosistema, conocidos por su inteligencia y herramientas de refactorización de código.
  • Eclipse: Es un IDE muy conocido y de código abierto, especialmente popular en el mundo del desarrollo empresarial. Es famoso por su excelente soporte para Java. También se utiliza para C/C++, PHP y otros lenguajes a través de plugins.
  • Xcode: Es el IDE oficial de Apple, indispensable para desarrollar software para cualquier dispositivo de Apple. Los lenguajes Utilizados son: Swift (el lenguaje moderno de Apple) y Objective-C (el lenguaje más antiguo). Se usa para crear aplicaciones para iOS, macOS, watchOS y tvOS.

Frameworks para backend y frontend

  • Frameworks de Backend
  • Express.js (Node.js): Es un framework minimalista y flexible para JavaScript (ejecutado en el entorno Node.js). Es extremadamente popular para construir APIs y aplicaciones web de forma rápida y sencilla.
  • Django (Python): Es un framework de "baterías incluidas" para Python. Ofrece una estructura muy completa y segura que incluye un panel de administración, sistema de autenticación y herramientas para interactuar con bases de datos (ORM), ideal para desarrollar aplicaciones complejas rápidamente.
  • Spring Boot (Java): Es el framework más popular en el ecosistema de Java para crear microservicios y aplicaciones web robustas y de alto rendimiento. Es muy utilizado en el mundo empresarial por su seguridad y escalabilidad.
  • Frameworks de Frontend
  • React (JavaScript): Desarrollado por Meta (Facebook), es una biblioteca (aunque comúnmente se le llama framework) para construir interfaces de usuario basadas en componentes. Es increíblemente popular por su flexibilidad y su gran ecosistema, que incluye herramientas como Redux para el manejo del estado.
  • Angular (TypeScript): Desarrollado por Google, es un framework completo y robusto para TypeScript (una versión de JavaScript con tipado estático). Ofrece una estructura muy opinionada, ideal para grandes aplicaciones empresariales (SPAs o Single Page Applications)
  • Vue.js (JavaScript): Conocido por su curva de aprendizaje suave y su excelente documentación. Es un framework progresivo, lo que significa que puedes usarlo para controlar una pequeña parte de una página o para construir una aplicación completa desde cero. Combina lo mejor de React y Angular en un paquete más ligero.
